拓扑排序是一种**只适用于有向无环图**（DAG）的排序方式，用来解决存在“先后依赖关系”、不能乱序、需要安排顺序的问题，如先修课程安排、任务调度、编译顺序、事件依赖系统等。

### 简介

拓扑排序是一种**只适用于有向无环图**（DAG）的排序方式，用来解决存在“先后依赖关系”、不能乱序、需要安排顺序的问题，如先修课程安排、任务调度、编译顺序、事件依赖系统等。

它把图中的所有节点排成一个线性序列，使得 每条有向边 u → v 中，u 都排在 v 前面。

注意：必须是无环图才有排序结果！有环会在排序过程被检查出。

### 实现方式

有两种经典实现方式：

#### 入度法（BFS思想）

本质是Kahn 算法。

**步骤**：

1. 统计每个点的入度
2. 入度为 0 的先入队
3. 不断出队并删除它的边
4. 新入度为 0 的继续入队

**特点**：逻辑清晰简单，适合面试场景，可以用来判断是否有环（即能否取完所有的点）

**代码**：（课程表II、Kahn 算法典型题）

```java
import java.util.*;

class Solution {
    /**
     * 拓扑排序 —— BFS（入度表方法 / Kahn算法）
     *
     * @param n     节点数量，默认节点编号 0~n-1
     * @param edges 边 from -> to  表示 from 必须先于 to
     */
    public List<Integer> topologicalSort(int n, int[][] edges) {

        // 1. 建图（邻接表表示）
        List<List<Integer>> graph = new ArrayList<>();
        for (int i = 0; i < n; i++) {
            graph.add(new ArrayList<>());
        }

        // 2. 统计每个节点的入度
        int[] indegree = new int[n];

        for (int[] e : edges) {
            int from = e[0];
            int to = e[1];

            graph.get(from).add(to); // 建边
            indegree[to]++;          // 入度 +1
        }

        // 3. 把所有 入度为 0 的点先放入队列
        Queue<Integer> queue = new LinkedList<>();
        for (int i = 0; i < n; i++) {
            if (indegree[i] == 0) {
                queue.offer(i);
            }
        }

        List<Integer> res = new ArrayList<>();

        // 4. BFS
        while (!queue.isEmpty()) {

            // 取出一个当前没有依赖的课程
            int cur = queue.poll();
            res.add(cur);

            // 遍历它的所有“后继节点”
            for (int next : graph.get(cur)) {

                // 删除 cur -> next 的依赖（入度减一）
                indegree[next]--;

                // 入度为 0，说明现在可以学
                if (indegree[next] == 0) {
                    queue.offer(next);
                }
            }
        }

        // 5. 如果能取完 → 无环，合法拓扑序
        //    如果没取完 → 有环（死锁依赖）
        if (res.size() != n) {
            return new ArrayList<>(); // 返回空表示有环
        }

        return res;
    }
}
```

#### DFS法（逆后序）

本质是逆后序遍历。

**步骤**：

1. 统计每个点的入度
2. 入度为 0 的先入队
3. 不断出队并删除它的边
4. 新入度为 0 的继续入队

**代码**：（课程表I、深度优先拓扑序）

```java
import java.util.*;

class Solution2 {

    List<List<Integer>> graph; // 邻接表
    int[] state;               // 0=未访问 1=访问中 2=已完成
    boolean hasCycle = false;  // 是否存在环
    List<Integer> res = new ArrayList<>();

    public List<Integer> topologicalSort(int n, int[][] edges) {

        // 1. 建图
        graph = new ArrayList<>();
        for (int i = 0; i < n; i++) {
            graph.add(new ArrayList<>());
        }

        for (int[] e : edges) {
            graph.get(e[0]).add(e[1]);
        }

        // 2. 初始化访问状态
        state = new int[n];

        // 3. 对所有点进行 DFS（可能不连通）
        for (int i = 0; i < n; i++) {
            if (state[i] == 0) {
                dfs(i);
            }
        }

        // 4. 如果有环 → 无拓扑序
        if (hasCycle) {
            return new ArrayList<>();
        }

        // 5. 后序遍历结果反转即为拓扑排序
        Collections.reverse(res);
        return res;
    }

    private void dfs(int node) {

        // 如果已经检测到有环，直接返回，减少递归
        if (hasCycle) return;

        // 标记为 "访问中"
        state[node] = 1;

        // 访问所有后继节点
        for (int next : graph.get(node)) {

            if (state[next] == 0) {
                // 没访问 → 递归
                dfs(next);
            } 
            else if (state[next] == 1) {
                // 遇到“访问中” → 回边 → 有环
                hasCycle = true;
                return;
            }
        }

        // 标记为 "已完成"
        state[node] = 2;

        // ★★★ 关键：后序位置加入结果
        res.add(node);
    }
}
```

>个人理解，仅供参考：
>
>如果代入经典的“课程表先后修的问题”，我认为可以把这里的dfs看作“去学习”。
>
>输入如
>
>>n = 4
>>edges = [
>>  [0, 1],
>>  [0, 2],
>>  [1, 3],
>>  [2, 3]
>>]
>
>也就是说，最开始我们先构建图，整理成一个这样的列表： 学习index之前需要修完list.get（index）的全部课程
>
>即：
>
>>0 ： 1， 2
>>
>>1 ： 3
>>
>>2 ： 3
>
>那么我们依次遍历，dfs(0)，去学习0，当发现其他没学就dfs(1)和dfs(2)，然后在这个过程发现3没学再去学3，非常像我们正常思考这个问题的逻辑，只要在这个过程发现“我学习x的时候发现我需要学习x？？说明有环！”。然后就会发现我们这样“学习”的顺序的倒序就是我们真正应该学习的顺序。
>
