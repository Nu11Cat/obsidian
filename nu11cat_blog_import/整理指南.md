## 🎯 第一阶段：基础标准化整理

### 1.  统一Frontmatter格式

**当前格式**：
```yaml
---
title: Claude Code
tags: [OpenClaw, AI, Tools]
categories: [AI编程与应用]
toc: true
---
```

**建议优化为Obsidian友好格式**：
```yaml
---
title: Claude Code 入门与使用指南
date: 2024-04-17  # 建议从文件名或内容提取
updated: 2024-04-17
tags: [AI, 开发工具, Claude, OpenClaw]
categories: AI编程与应用
toc: true
publish: false  # 标记是否为发布状态
aliases: [Claude Code使用教程, Claude Code CLI]  # 方便链接
---
```

### 2.  清理博客特定语法

- **移除`<!-- more -->`分隔符**：Obsidian不需要这个摘要分割
- **修复相对链接**：将`[链接文字](/posts/xxx)`改为Obsidian Wiki链接`[[笔记名称|链接文字]]`
- **标准化代码块格式**：统一使用```语言名 格式

### 3.  批量重命名优化

**当前文件名**：`什么是AI Agent.md`
**建议改为**：`20240417-什么是AI-Agent.md`（添加日期前缀+标准化空格替换）

## 📌 快速自动化处理建议

### 批量处理脚本（可直接运行）

1.  **批量添加日期前缀**：
    ```bash
    for file in $(find . -name "*.md"); do
      # 从文件修改时间提取日期
      date=$(stat -c "%y" "$file" | cut -d' ' -f1 | tr -d '-')
      mv "$file" "$(dirname "$file")/${date}-$(basename "$file")"
    done
    ```

2.  **批量移除`<!-- more -->`**：
    ```bash
    find . -name "*.md" -exec sed -i '/<!-- more -->/d' {} \;
    ```

### 手动处理优先级

1.  先处理高频访问的分类（如Java/数据库）
2.  优先处理有代码示例的实用文章
3.  最后整理理论性较强的文章
