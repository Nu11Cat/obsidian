## 通配符

**常用通配符**

rm *.log      # 删除所有.log结尾的文件，*匹配任意数量任意字符（包括空字符）

ls file?.txt  # 匹配 file1.txt、fileA.txt 等 ，？匹配单个任意字符

ls file[123].txt  # 匹配 file1.txt、file2.txt、file3.txt，[]匹配括号内任意一个字符

ls file[!a-z].txt  # 排除字母命名的文件，[!]或[^]匹配不在括号内的字符

cp file.{txt,bak}  # 同时操作 file.txt 和 file.bak，{}生成多个匹配组合

## 进程管理

**查看进程/系统资源使用情况**

ps 查看当前进程快照

- ps aux       # 查看所有用户的所有进程（详细）  
- ps -ef       # 查看全格式进程列表（含父进程ID）  
- ps -eLf      # 查看进程及线程（L显示线程）
- ps aux | grep 进程名 # 模糊匹配进程名（常用，但会包含grep自身）

top 实时动态查看进程（按 `q`退出）

- top              # 默认按CPU占用排序  
- top -p PID   # 监控指定PID的进程

----------

**终止线程**

kill -15 PID，请求进程正常退出

kill -9 PID，强制终止，立即杀死进程

pkill 进程名          # 按名称终止（如 `pkill nginx`）  

killall 进程名       # 终止所有同名进程（如 `killall java`）

---

**查看后台任务**

jobs -l，l 显示任务详情

-----

**后台任务调度**

fg %任务编号，后台任务调到前台执行

bg %任务编号，将停止的后台任务在后台继续执行

-----

**进程有哪些状态**

1. **运行（Running）** - 符号 `R`进程正在执行或就绪（等待 CPU 调度）。
2. **可中断睡眠（Interruptible Sleep）** - 符号 `S`进程在等待某个事件（如 I/O 完成），可以被信号唤醒。
3. **不可中断睡眠（Uninterruptible Sleep）** - 符号 `D`进程在等待硬件条件（如磁盘 I/O），**不能被信号中断**，常见于内核态操作。
4. **僵尸（Zombie）** - 符号 `Z`进程已终止，但其父进程尚未回收资源（需手动清理或等待父进程退出）。
5. **停止（Stopped）** - 符号 `T`进程被信号（如 `SIGSTOP`）暂停，或正在被调试器跟踪。
6. **跟踪（Tracing stop）** - 符号 `t`进程被调试器（如 `gdb`）暂停跟踪。
7. **死亡（Dead）** - 符号 `X`进程完全终止（极少在 `ps`中看到）。

## Java相关

**启动 jar 包程序**

java -jar app.jar

----

**如何后台运行一个Java程序，并且将输出重定向到日志文件**

nohup java -jar your_app.jar > output.log 2>&1 &

- nohup 保证终端关闭后程序仍运行
- \> output.log 将标准输出（`stdout`）重定向到 `output.log`文件
- 2>&1 将错误输出（`stderr`）合并到标准输出
- & 让程序在后台运行

------

**如何传递系统参数或JVM参数给Java程序？**

java -Xms512m -Xmx1024m -XX:+UseG1GC -jar app.jar

- `-Xms`：初始堆内存（如 `-Xms512m`）。
- `-Xmx`：最大堆内存（如 `-Xmx1024m`）。
- `-XX:+UseG1GC`：启用 G1 垃圾回收器。
- `-XX:MaxMetaspaceSize=256m`：限制元空间大小。

----

**有哪些JDK自带的Linux排查工具**

jps -l，查看当前用户的 Java 进程

jstack PID，抓取 Java 进程的线程堆栈（排查死锁、卡顿）

jstat，监控 JVM 统计信息（GC、类加载等）

jmap，生成堆内存转储（分析内存泄漏）
