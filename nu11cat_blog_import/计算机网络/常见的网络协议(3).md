## **IP**

IP（Internet Protocol，网际协议） 属于网络层的协议，主要作用是定义数据包的格式、对数据包进行路由和寻址，以便它们可以跨网络传播并到达正确的目的地。

### **IP地址，IPv4 和 IPv6**

IP 地址是分配给网络接口的逻辑标识，用于在网络中定位通信端点，在特定网络范围内具有唯一性。

IPv4 和 IPv6是两个不同版本的IP协议：

1. IPv4 使用 32 位地址，以点分十进制表示，最多约 42 亿个地址，已接近枯竭；而 IPv6 使用 128 位地址，以冒号分隔的十六进制表示，地址空间极其庞大。
2. IPv4 报文头字段较多，长度可变；IPv6 采用固定长度的简化头部，并将扩展功能通过扩展头实现，更利于高速转发。
3. IPv4 中广泛使用 NAT 解决地址不足；IPv6 不需要 NAT，每个设备可拥有唯一公网地址，避免了 IPv4 中因 NAT 带来的连接复杂、端到端通信受限等问题
4. IPv4 原生支持较弱，依赖额外配置；IPv6 在协议规范中定义了对 IPsec 的支持能力，为端到端加密和身份认证提供了基础，但在实际网络中是否启用仍取决于具体部署情况。

### **IP寻址流程**

1. 检查是否是同一网段：发送方通过子网掩码判断目标 IP 是否与自己处于同一网段。

   - 如果是：直接通过局域网 ARP 协议获取目标设备的 MAC 地址，完成通信。

   - 如果不是：将数据包交给默认网关（通常是路由器）转发。

1. 路由转发：如果目标 IP 不在本地网段，数据包会被交给路由器，路由器根据自己的路由表决定下一跳，将数据一跳一跳转发，直到到达目标网络。
2. 最终送达主机：到达目标网络后，最后一跳路由器使用 ARP 找到目标主机的 MAC 地址，完成局域网传输。

#### **NAT--- 地址转换技术**  

 把私有 IP 地址转换为公网 IP 地址，实现多设备共享一个公网 IP， 解决 IPv4 地址不足问题 

 依赖 IP，通常在路由器或防火墙设备实现   

优点：缓解了 IPv4 地址短缺的问题，隐藏内部网络 IP 地址，在一定程度上降低了被直接访问的风险，但 NAT 本身并不等同于安全机制。

缺点：破环端到端通信，并且因为维护映射表提高了网络配置的复杂性

### **IP 地址 vs MAC 地址**

首先，IP 地址是网络层的逻辑寻址机制，决定“去哪”；而 MAC 地址是链路层的物理标识机制，确定“交给谁”，二者属于不同层次的抽象，功能不重叠，没有“替代”和“比较”的意义

IP 地址：属于 网络层  ，用于标识网络中主机的逻辑地址，用来进行跨网络寻址和路由转发。可配置、可变，是可路由的，贯穿整个互联网。

MAC 地址：属MAC 地址属于数据链路层（在 TCP/IP 模型中对应网络接口层），仅在局域网范围内有效，用于链路级的数据转发。不参与跨网传输，通常不变，是局域网内的网络设备的唯一标识，负责在同一链路进行点对点通信。

## **ICMP**

ICMP（Internet Control Message Protocol） ，网际控制报文协议。

- ICMP 协议是一种面向无连接的协议，用于传输出错报告控制信息。
- 它属于网络层协议，主要用于在主机与路由器之间传递控制信息，包括**报告错误、交换受限控制和状态信息**等。
- 当遇到 IP 数据无法访问目标、IP 路由器无法按当前的传输速率转发数据包等情况时，会自动发送 ICMP 消息。
- 比如我们日常使用得比较多的 **ping**，就是基于 ICMP 的。

---

### **ping 的原理**

`ping` 就是向目标 IP 发送一个 ICMP Echo Request 报文，等待目标返回 ICMP Echo Reply 报文，然后计算两者的时间差（RTT），并统计丢包率与延迟。
