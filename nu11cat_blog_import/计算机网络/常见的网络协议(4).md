常见的网络接口层网络协议：ARP等。

<!-- more -->



## **ARP**

ARP（Address Resolution Protocol，地址解析协议）用于**在局域网内将网络层的 IP 地址解析为数据链路层的 MAC 地址**，是 IP 协议正常工作的基础支撑协议之一。

ARP 报文封装在数据链路层帧中传输，在 TCP/IP 模型中通常被归类为**网络接口层协议**（对应 OSI 模型的数据链路层）。它本身不负责数据转发，而是为 IP 层提供从逻辑地址到物理地址的映射能力。

------

### **工作原理**

当主机需要向目标 IP 地址发送数据，但本地 ARP 缓存中尚不存在该 IP 对应的 MAC 地址时，会执行以下流程：

1. **ARP 请求（广播）**
    主机在局域网内广播 ARP Request 报文，询问“哪个设备拥有该目标 IP 地址”。
2. **ARP 响应（单播）**
    局域网内所有主机都会接收该请求，其中 IP 地址匹配的主机会向请求方发送 ARP Reply 报文，返回自己的 MAC 地址。
3. **ARP 缓存**
    请求方收到响应后，会将 IP 地址与 MAC 地址的映射关系缓存到本地 ARP 表中，在缓存有效期内可直接使用该映射进行通信，避免重复广播。

通过 ARP，IP 数据包才能在局域网中被正确封装为以太网帧并完成最终的链路层传输。
