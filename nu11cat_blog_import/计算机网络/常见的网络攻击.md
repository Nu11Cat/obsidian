## **常见的网络攻击**

常见的网络攻击大致可以分为四类：

第一类是**应用层攻击**，主要针对网站与浏览器，例如 HTTP Flood、XSS 攻击和 CSRF 攻击，通过发送伪造的正常请求或注入恶意脚本来消耗资源或窃取数据。

第二类是**协议层攻击**，直接针对 TCP/UDP/IP 等传输协议本身，例如 TCP 重置攻击、UDP Flood 和 IP 欺骗，通过伪造报文或洪泛流量来干扰或中断通信。

第三类是**中间人攻击**，攻击者位于通信双方之间，对数据进行拦截、篡改或伪造，如 ARP 欺骗、DNS 劫持、Wi-Fi 嗅探或伪造证书进行 SSL 劫持。

第四类是**拒绝服务攻击（DoS/DDoS/DRDoS）**，通过大量恶意流量或反射放大流量使服务不可用，常见于大规模流量型攻击场景。

------

## **HTTP Flood**

应用层攻击。

HTTP Flood 是一种应用层 DDoS 攻击，攻击者伪装为正常用户，持续发送看起来合法的 HTTP 请求，从而大量占用服务器计算与并发资源，使网站响应变慢甚至崩溃。

特点：高隐蔽性；行为接近正常访问；带宽占用不一定大但服务器压力极大。

**应对方法**

- 行为识别与挑战机制（验证码、JS 校验）
- 接入 WAF/CDN，对异常请求进行限流与清洗

------

## **CSRF 攻击**

应用层攻击。

CSRF（跨站请求伪造）是指攻击者诱导已登录用户，在用户不知情的情况下向目标网站发起请求，利用浏览器会自动携带 Cookie/Session 的特性，冒充用户身份执行敏感操作，如转账或改密。

**应对方法**

- 使用 CSRF Token 并与会话绑定
- 校验 Referer/Origin 来源

------

## **XSS 攻击**

应用层攻击。

XSS（跨站脚本攻击）是指攻击者向网页中注入恶意脚本，使其在用户浏览器中执行，从而窃取敏感信息、劫持会话、执行未授权操作，甚至控制用户账户。

**XSS 攻击三种类型**

1. 存储型：恶意脚本存储在服务器端，影响面大且可长期存在
2. 反射型：脚本经 URL/表单立即被反射到页面，一次性攻击
3. DOM 型：完全发生在浏览器端，由前端脚本操作 DOM 时触发

**应对方法**

- 严格输出编码与转义
- 启用 CSP（内容安全策略）

------

## **TCP 重置攻击**

传输层攻击。

攻击者伪造带有 **RST 标志位** 的 TCP 报文，并冒充通信双方之一，在正确的连接信息和序列号范围内强制关闭连接，使正在进行的 TCP 会话被意外中断。

特点：隐蔽、成本低、对长连接和实时业务影响大。

**应对方法**

- 随机化 TCP 初始序列号、加强报文校验
- 使用 TLS/SSL，检测异常断连并确保数据完整性

------

## **UDP Flood**

传输层攻击。

UDP Flood 是通过向目标发送大量 UDP 数据报造成资源消耗，使服务器不断处理无效端口、丢弃数据或尝试返回 ICMP 报文，从而耗尽 CPU、内存或带宽资源。

**应对方法**

- 设置 UDP 流量限速与过滤策略
- 使用抗 DDoS/清洗服务吸收大流量

------

## **IP 欺骗**

网络层攻击。

IP 欺骗是伪造报文源 IP 地址的攻击方式，使接收方误以为数据来自受信任主机，从而隐藏真实身份、绕过基于 IP 的访问控制或配合放大/反射型攻击使用。

**应对方法**

- 入口/出口过滤，禁止伪造源地址
- 启用反欺骗机制（如 uRPF）

------

## **中间人攻击**

中间人攻击是攻击者插入到通信双方之间，对数据进行拦截、监听或篡改，从而窃取敏感信息或伪造通信内容。

**常见方式**

- Wi-Fi 嗅探
- ARP 欺骗
- DNS 劫持
- 伪造证书进行 SSL 劫持

**应对方法**

- 全站启用 HTTPS 与合法数字证书
- 开启 HSTS，强制使用加密连接

------

## **DoS / DDoS / DRDoS**

- **DoS**：单一攻击源导致服务不可用
- **DDoS**：多台受控主机同时发起攻击，更难溯源与阻断
- **DRDoS**：反射+放大型 DDoS，利用第三方服务器放大发往受害者的流量
