## 快速开始

### 准备工作

首先你要先安装Node.js和git

Node.js官网：https://nodejs.org/

git官网：[Git](https://git-scm.com/?hl=zh-cn)

安装完成后，打开命令行验证：

```bash
node -v
npm -v
```

然后安装Hexo命令行工具

```bash
npm install -g hexo-cli
```

验证安装

```bash
hexo -v
```

### 初始化博客目录

选择你想要存放博客的文件夹

```
mkdir D:\blog
cd D:\blog
hexo init
```

`hexo init` 会生成默认目录结构，并自动安装依赖。

然后安装依赖

```bash
npm install
```

### 启动Hexo

```bash
hexo clean
hexo s
```

然后打开浏览器访问

```
http://localhost:4000
```

就能看到Hexo的首页

### 安装Icarus主题

在博客根目录：

```bash
git clone https://github.com/ppoffice/hexo-theme-icarus themes/icarus
```

然后打开根目录下的`_config.yml`：

搜索theme，把值换成我们的主题icarus

```
theme: icarus
```

然后安装 Icarus 推荐依赖

```bash
npm install hexo-renderer-inferno hexo-renderer-marked hexo-renderer-stylus hexo-generator-searchdb --save
```

### 启动Icarus

```
hexo clean
hexo s
```

启动并查看效果

```
http://localhost:4000
```

你会看到 Icarus 默认首页，卡片式文章列表。

### 创建文章

继续在根目录下

```text
hexo new post "我的第一篇博客"
```

然后重新启动就能看到新文章的卡片

### 可能的问题

如果你没有遇到问题最好，但是你可能遇到Hexo版本与Icarus版本不兼容的问题，这里为你提供一个解决方法：

降低Hexo版本，安装7.x版本的Hexo，重新安装依赖，重新启动：

```bash
npm uninstall -g hexo-cli
npm install -g hexo-cli@4.0.0
npm install --save bulma-stylus@0.8.0 hexo@^7.1.1 hexo-component-inferno@^3.1.2 hexo-util@^3.2.0 semver@^7.5.4
hexo clean
hexo s
```

如果有其他更多的问题可以查看文章末的”可能的问题“

## 配置Hexo

配置Hexo主要修改`_config.yml`

各项配置的意义参考：

```yaml
# Hexo Configuration
## Docs: https://hexo.io/docs/configuration (链接7)
## Source: https://github.com/hexojs/hexo (链接1)

# Site - 网站信息设置
title: Nu11Cat # 网站标题
subtitle: Nu11Cat的技术博客网站 # 网站副标题
description: Nu11Cat的技术博客网站 # 网站描述，主要用于SEO
keywords: 技术博客 # 网站关键词，用于SEO

permalink: :year/:month/:day/:title/ # 文章永久链接的格式，例如：2023/10/27/hello-world/
permalink_defaults: # 永久链接中各部分的默认值（通常留空）
pretty_urls: # 美化URL设置

filename_case: 0 # 转换文件名大小写：0（不转换），1（转为小写），2（转为大写）
render_drafts: false # 是否渲染草稿（drafts文件夹中的文章）
post_asset_folder: false # 是否为每篇文章创建一个同名资源文件夹来存放图片等资源
relative_link: false # 是否使用相对链接（通常用于本地文件系统）
future: true # 是否发布带有未来日期的文章
syntax_highlighter: highlight.js # 代码语法高亮引擎（highlight.js 或 prismjs）
highlight: # highlight.js 相关配置
  line_number: true # 是否显示行号
  auto_detect: false # 是否自动检测代码语言
  tab_replace: '' # 用于替换制表符的字符串（例如用空格替换）
  wrap: true # 是否用<figure>和<figcaption>包裹代码块
  hljs: false # 是否使用hljs-*类名进行高亮（为了兼容性）
prismjs: # prismjs 相关配置（如果syntax_highlighter设为prismjs则生效）
  preprocess: true # 是否预处理代码（适用于Node.js环境）
  line_number: true # 是否显示行号
  tab_replace: '' # 用于替换制表符的字符串

# Home page setting - 首页设置
# path: 博客索引页的根路径（默认 = ''）
# per_page: 每页显示的文章数量（0 = 禁用分页）
# order_by: 文章排序规则（默认按日期降序排列）
index_generator:
  path: ''
  per_page: 10
  order_by: -date

# Category & Tag - 分类和标签设置
default_category: uncategorized # 默认分类（当文章未指定分类时使用）
category_map: # 分类别名映射（例如：'编程': 'coding'）
tag_map: # 标签别名映射（例如：'JavaScript': 'js'）

# Metadata elements - HTML元数据元素
## 参考：https://developer.mozilla.org/en-US/docs/Web/HTML/Element/meta (链接3)

## https://momentjs.com/docs/ (链接9)

time_format: HH:mm:ss # 时间显示格式（时:分:秒）
## updated_option 控制当文章Front Matter中未提供updated（更新日期）时的取值逻辑：'mtime'（使用文件修改时间），'date'（使用创建日期），'empty'（留空）
updated_option: 'mtime'

# Pagination - 分页设置
## 将 per_page 设置为 0 以禁用分页
per_page: 10 # 每页显示的文章数量
pagination_dir: page # 分页目录名称（例如：第2页的URL为 /page/2/）

# Include / Exclude file(s) - 包含/排除文件设置
## include:/exclude: 选项仅适用于 'source/' 文件夹
include: # 明确包含的文件或文件夹（即使它们以_开头或为隐藏文件）
exclude: # 排除的文件或文件夹（不进行处理）
ignore: # 忽略的文件或文件夹（适用于所有文件夹，包括themes/）

# Extensions - 扩展功能
## Plugins: https://hexo.io/plugins (链接4) - 社区插件列表
## Themes: https://hexo.io/themes (链接5) - 社区主题列表

# Deployment - 部署设置
## Docs: https://hexo.io/docs/one-command-deployment (链接6)
deploy:
  type: '' # 部署器类型，例如：'git', 'rsync'等。需要先通过npm安装对应的hexo-deployer-*插件
  # 根据你选择的部署器类型（type），在此处添加相应的配置项（如repo, branch等）
```

## 配置Icarus主题

首先确定一点，你的根目录应该存在`_config.icarus.yml` 和 `\themes\icarus`文件夹，如果没有那就是你前面搞错了，如果没有特殊说明，此部分的配置都是`_config.icarus.ym`l内的

### Logo和Favicon

站点的logo会显示在导航栏和页脚

```
logo: /img/logo.svg
```

或者设置成文字

```
logo:
    text: My Beautiful Site
```

**Favicon**

```
head:
    favicon: /img/favicon.svg
```

### 导航栏/页脚

顶部导航栏

```yaml
navbar:
    # 导航栏菜单项
    menu:
        Home: /
        Archives: /archives
        Categories: /categories
        Tags: /tags
        About: /about
    # 导航栏右侧的链接
    links:
        GitHub: 'https://github.com'
        Download on GitHub:
            icon: fab fa-github
            url: 'https://github.com/ppoffice/hexo-theme-icarus'
```

也可以使用FontAwesome图标来作为纯文字链接的替换，格式如下：

```yaml
<链接名>:
    icon: <FontAwesome图标的class名>
    url: <链接URL>
```

**页脚**

```jaml
footer:
    links:
        Creative Commons:
            icon: fab fa-creative-commons
            url: 'https://creativecommons.org/'
        Attribution 4.0 International:
            icon: fab fa-creative-commons-by
            url: 'https://creativecommons.org/licenses/by/4.0/'
        Download on GitHub:
            icon: fab fa-github
            url: 'https://github.com/ppoffice/hexo-theme-icarus'
```

也可以展示文字：

```yaml
footer:
    copyright: 用💖发电
```

### 文章阅读时间

开启之后会显示章字数统计以及预计阅读时间。

```yaml
article:
    readtime: true
```

### 文章更新时间

若要显示文章的更新时间，请在文章的front_matter中设置`updated`时间：

```
post.md

title: Icarus快速上手
updated: 2020-04-01 00:00:00
---
Post content...
```

然后，将主题配置文件的`article`部分的`update_time`设置为`true`：

```yaml
article:
    update_time: true
```

你也可以将`update_time`设置为`false`来隐藏所有文章的更新时间，或设置为`auto`而在文章的更新时间 与发布时间相同时隐藏更新时间。

### 文章许可协议

你可以在你的文章/页面的底部展示你的作品的使用许可，许可链接可以是文字或者图标。 这里的配置与导航栏或者页脚的`links`配置一致：

```yaml
article:
    # 文章许可协议
    licenses:
        Creative Commons:
            icon: fab fa-creative-commons
            url: 'https://creativecommons.org/'
        'CC BY-NC-SA 4.0': 'https://creativecommons.org/licenses/by-nc-sa/4.0/'
```

### 侧边栏跟随

设置`sidebar`中某个侧边栏的`sticky`为`true`来让它的位置固定而不跟随页面滚动。

```yaml
sidebar:
    left:
        sticky: false
    right:
        sticky: true
```

### 布局与组件

Icarus支持三种布局，也就是三栏布局，两栏布局，单页布局，具体效果见下：

[ppoffice/hexo-theme-icarus: A simple, delicate, and modern theme for the static site generator Hexo.](https://github.com/ppoffice/hexo-theme-icarus?tab=readme-ov-file#flexible-theme-configuration)

建议首页采用三栏，文章采用两栏，以及调整”文章“部分的宽度，毕竟谁也不想阅读夹在缝隙的文章

我们需要修改`_config.icarus.yml`，把widgets（组件）放在左右两边:

然后调整整体宽度

在themes\icarus\include\style目录下，找到responsive.styl，将widescreen样式中的

```yaml
+widescreen()
    .is-1-column .container, .is-2-column .container
        max-width: $desktop - 2 * $gap
        width: $desktop - 2 * $gap
```

改成

```yaml
+widescreen()
    .is-1-column .container, .is-2-column .container
        max-width: $widescreen  - 2 * $gap
        width: $widescreen  - 2 * $gap
```
