`HashSet` 与 `LinkedHashSet` 都是 `Set` 实现，底层不单独维护另一套哈希结构，而是委托给 `HashMap` 与 `LinkedHashMap`。读这两类源码的核心是：**集合语义如何映射到 Map 的键值语义**。

以下代码摘录自 **OpenJDK 8u**，泛型为阅读方便已补全。

## HashSet 简介

`HashSet` 继承 `AbstractSet`，内部持有一个 `HashMap<E,Object>`。集合中的每个元素对应 `Map` 的一个键，值统一为占位对象 `PRESENT`，从而用 `put` / `remove` 的返回值表达「是否新增成功」「是否删除成功」。

迭代顺序与 `HashMap` 一致，不保证稳定。允许一个 `null` 元素。非线程安全。

-----------

## HashSet 源码解析

### 核心字段与占位值

```java
public class HashSet<E> extends AbstractSet<E>
        implements Set<E>, Cloneable, java.io.Serializable {

    private transient HashMap<E, Object> map;

    private static final Object PRESENT = new Object();
```

### 构造与底层 Map

无参构造直接创建默认参数的 `HashMap`：

```java
public HashSet() {
    map = new HashMap<>();
}

public HashSet(int initialCapacity, float loadFactor) {
    map = new HashMap<>(initialCapacity, loadFactor);
}
```

供 `LinkedHashSet` 使用的包内构造：第三个参数 `dummy` 仅用于区分重载，实际创建的是 `LinkedHashMap`：

```java
HashSet(int initialCapacity, float loadFactor, boolean dummy) {
    map = new LinkedHashMap<>(initialCapacity, loadFactor);
}
```

### add、contains、remove

```java
public boolean add(E e) {
    return map.put(e, PRESENT) == null;
}

public boolean contains(Object o) {
    return map.containsKey(o);
}

public boolean remove(Object o) {
    return map.remove(o) == PRESENT;
}
```

- `add`：`put` 返回 `null` 表示原先无该键，即集合中新增成功。
- `remove`：原先存在该键时 `remove` 返回旧值 `PRESENT`。

### 迭代与规模

```java
public Iterator<E> iterator() {
    return map.keySet().iterator();
}

public int size() {
    return map.size();
}
```

`HashSet` 的迭代实际是遍历 backing `HashMap` 的 `keySet()`，因此迭代代价与 `HashMap` 容量和元素数相关；文档中强调不宜把初始容量设得过大，以免拖慢遍历。

## LinkedHashSet 简介

`LinkedHashSet` 继承 `HashSet`，未再声明新的字段。通过调用父类包内三参构造，使底层 `map` 为 **`LinkedHashMap`**，从而获得**插入顺序**可预测的迭代顺序。

-----------

## LinkedHashSet 源码解析

### 构造器全部走 `HashSet(int, float, boolean)`

```java
public class LinkedHashSet<E> extends HashSet<E>
        implements Set<E>, Cloneable, java.io.Serializable {

    public LinkedHashSet(int initialCapacity, float loadFactor) {
        super(initialCapacity, loadFactor, true);
    }

    public LinkedHashSet(int initialCapacity) {
        super(initialCapacity, .75f, true);
    }

    public LinkedHashSet() {
        super(16, .75f, true);
    }

    public LinkedHashSet(Collection<? extends E> c) {
        super(Math.max(2 * c.size(), 11), .75f, true);
        addAll(c);
    }
}
```

从集合构造时，`Math.max(2*c.size(), 11)` 与 `HashSet` 从集合构造时的容量估算不同，这是为减少插入过程中的扩容次数而做的取舍。

### 与 HashSet 的方法关系

`LinkedHashSet` 未覆写 `add`、`remove`、`contains`，逻辑与 `HashSet` 完全相同，顺序语义完全由底层 `LinkedHashMap` 的链表维护。

JDK8 中覆写 `spliterator()`，在特征位上增加 `ORDERED`，与插入顺序迭代一致。

## 常见问题

### 1. HashSet 里 value 为什么用同一个 PRESENT？

所有键共享同一常量引用即可，不需要为每个元素分配不同值；`remove` 时只需判断返回值引用是否等于 `PRESENT`。

### 2. LinkedHashSet 能否配置成访问顺序？

公开 API 未暴露 `LinkedHashMap` 的 `accessOrder` 构造参数；默认即为插入顺序。若需要访问顺序，应直接使用 `LinkedHashMap` 或自行封装。

### 3. 线程安全吗？

两者均非线程安全，多线程修改需 `Collections.synchronizedSet` 或并发集合方案。

-----------

## 总结

`HashSet` 与 `LinkedHashSet` 本质是 **`HashMap` / `LinkedHashMap` 的键视图**，源码量小但把 Map 与 Set 的对应关系写得很清楚。

需要无序去重集合用 `HashSet`，需要稳定插入顺序用 `LinkedHashSet`，需要访问顺序或复杂淘汰策略应下沉到 `LinkedHashMap` 自行控制。
