## LinkedList简介

`LinkedList`是Java集合`Collection`根接口下的一个实现类，同时实现了`List`接口和`Deque`接口。既可以看作是一个顺序容器，也可以看作一个队列(`Queue`)，也可以看作一个栈(`Stack`)。它是线程不安全的。LinkedList 允许元素为 `null`，而且允许多个 `null`。

>Java官方声明不建议使用`Stack`类（基于设计不合理的vector），而且Java没有`Queue`类（它只是一个接口名字）。
>
>当你需要使用栈或者队列时，可以考虑使用`LinkedList`，但是首选实际上是`ArrayDeque`，它的性能更好。

## 源码解析

### 底层数据结构

`LinkedList`底层是通过双向链表实现的，双向链表的每个节点用内部类`Node`表示。`LinkedList`通过`first`和`last`引用分别指向链表的第一个和最后一个元素。注意这里没有所谓的哑元，当链表为空的时候`first`和`last`都指向`null`。

私有内部类`Node`：

```java
private static class Node<E> {
    E item;
    Node<E> next;
    Node<E> prev;

    Node(Node<E> prev, E element, Node<E> next) {
        this.item = element;
        this.next = next;
        this.prev = prev;
    }
}
```

```java
transient int size = 0;

/**
 * 指向第一个节点的指针。
 * 不变式（Invariant）：
 * (first == null && last == null) ||
 * (first.prev == null && first.item != null)
 */
transient Node<E> first;

/**
 * 指向最后一个节点的指针。
 * 不变式（Invariant）：
 * (first == null && last == null) ||
 * (last.next == null && last.item != null)
 */
transient Node<E> last;
```

### 构造函数

1. 无参构造，什么都没干。因为LinkedList是链表结构，节点是按需创建的，所以不需要像`ArrayList`那样初始化。
2. 传入集合的构造方法，先调用`this();`复用无参构造，再使用`addAll(c);`统一入口而不是再自己遍历，避免逻辑重复带来Bug隐患。
3. 对于传入集合c的`NullPointerException`判断，这段代码在`addAll(c);`中，通过它统一对`null`进行保护。（是对传入集合c判断，不是对LinkedList判断。LinkedList允许元素为`null`）

>`addAll(c);`具体内容见下方 #addAll()

```java
/**
 * 构造一个空列表。
 */
public LinkedList() {
}

/**
 * 构造一个列表，其中包含指定集合中的元素，
 * 顺序为该集合迭代器返回元素的顺序。
 *
 * @param  c 其元素将被放入此列表的集合
 * @throws NullPointerException 如果指定的集合为 null
 */
public LinkedList(Collection<? extends E> c) {
    this();
    addAll(c);
}
```

### remove(Object o)

删除第一次出现的元素:

1. `unlink`中的`final`表示这些变量是快照，在整个方法执行期间不会被修改，保证代码逻辑安全。另外`final`在方法局部变量经常表示“只读”，也算是提高可读性的手段之一。

```java
/**
 * 如果存在，则从此列表中移除指定元素的第一次出现。
 * 如果此列表不包含该元素，则列表不发生变化。
 * 更正式地说，移除具有最小索引 {@code i} 的元素，使得
 * <tt>(o==null ? get(i)==null : o.equals(get(i)))</tt> 成立
 * （如果此类元素存在）。如果此列表包含指定元素
 * （或等价地说，此调用导致列表发生了变化），则返回 {@code true}。
 *
 * @param o 如果存在，将从此列表中被移除的元素
 * @return {@code true} 如果此列表包含指定的元素
 */
public boolean remove(Object o) {
    if (o == null) {
        for (Node<E> x = first; x != null; x = x.next) {
            if (x.item == null) {
                unlink(x);
                return true;
            }
        }
    } else {
        for (Node<E> x = first; x != null; x = x.next) {
            if (o.equals(x.item)) {
                unlink(x);
                return true;
            }
        }
    }
    return false;
}

/**
 * 断开非空节点 x 与链表的连接。
 */
E unlink(Node<E> x) {
    // assert x != null;
    final E element = x.item;
    final Node<E> next = x.next;
    final Node<E> prev = x.prev;

    if (prev == null) { // 第一个元素
        first = next;
    } else {
        prev.next = next;
        x.prev = null;
    }

    if (next == null) { // 最后一个元素
        last = prev;
    } else {
        next.prev = prev;
        x.next = null;
    }

    x.item = null; // 方便 GC 回收
    size--;
    modCount++;
    return element;
}
```

### removeFirst()

这个还是很简单的，没什么好说的。

```java
/**
 * 移除并返回此列表中的第一个元素。
 *
 * @return 此列表中的第一个元素
 * @throws NoSuchElementException 如果此列表为空
 */
public E removeFirst() {
    final Node<E> f = first;
    if (f == null)
        throw new NoSuchElementException();
    return unlinkFirst(f);
}

/**
 * 断开非空的首节点 f 与链表的连接。
 */
private E unlinkFirst(Node<E> f) {
    // assert f == first && f != null;
    final E element = f.item;
    final Node<E> next = f.next;
    f.item = null;
    f.next = null; // 便于 GC 回收
    first = next;
    if (next == null)
        last = null;
    else
        next.prev = null;
    size--;
    modCount++;
    return element;
}
```

### add()

1. add(E e) ，在末尾添加一个元素。O(1)，

2. add(int index, E element)，在指定位置插入一个元素。如果插入位置刚好是头部/末尾，那么就是O(1)，否则O(n)，因为它需要遍历到指定位置。

   - 非尾部插入：

   1. 找到 目标位置节点 `node(index)`
   2. 调用 `linkBefore(element, node)`，在目标节点前插入新节点
   3. 新节点会把原节点和前驱节点连接起来

3. node(int index)方法中，因为链表是双向的，所以这里有一个判断来决定是从前还是从后开始找。（但是也一定比ArrayList慢）

>checkPositionIndex(index); 方法见上方 # Positional Access方法

```java
/**
 * 将指定元素追加到此列表的末尾。
 *
 * <p>此方法等价于 {@link #addLast}。
 *
 * @param e 要追加到此列表的元素
 * @return {@code true} （由 {@link Collection#add} 指定）
 */
public boolean add(E e) {
    linkLast(e);
    return true;
}

/**
 * 将 e 链接为最后一个元素。
 */
void linkLast(E e) {
    final Node<E> l = last;
    final Node<E> newNode = new Node<>(l, e, null);
    last = newNode;
    if (l == null)
        first = newNode;
    else
        l.next = newNode;
    size++;
    modCount++;
}

```

```java
/**
 * 在此列表的指定位置插入指定元素。
 * 将当前位于该位置的元素（如果有）以及之后的所有元素向右移动（索引加一）。
 *
 * @param index 要插入指定元素的位置索引
 * @param element 要插入的元素
 * @throws IndexOutOfBoundsException {@inheritDoc}
 */
public void add(int index, E element) {
    checkPositionIndex(index);

    if (index == size)
        linkLast(element);
    else
        linkBefore(element, node(index));
}

/**
 * 返回指定元素索引处的（非空）节点。
 */
Node<E> node(int index) {
    // assert isElementIndex(index);

    if (index < (size >> 1)) {
        Node<E> x = first;
        for (int i = 0; i < index; i++)
            x = x.next;
        return x;
    } else {
        Node<E> x = last;
        for (int i = size - 1; i > index; i--)
            x = x.prev;
        return x;
    }
}

```

### addAll()

1. addAll(Collection<? extends E> c)，直接调用addAll(size, c)表示尾部添加。
2. addAll(int index, Collection<? extends E> c)，表示将指定集合中的所有元素插入到此列表的指定位置。
   1. 检查索引合法性
   2. 把集合转化成数组
   3. 定位插入位置，其中`prev`表示前驱节点，`succ` 是后继节点。新节点会插入在它们之间。
   4. 遍历数组，每个元素生成一个新节点;新节点的 prev 指向 `pred`，next 暂时是 `null`;更新前驱节点的 next 指向新节点;把 `pred` 移动到新节点，准备下一个元素链接。
   5. 连接新增节点与后继节点
   6. 更新size 和 modCount


>1. @SuppressWarnings("unchecked") E e = (E) o;是什么意思？
>
>`o` 是 `Object` 类型（因为 `c.toArray()` 返回的是 `Object[]`）
>
>`(E) o` 强制类型转换为泛型类型 `E`
>
>`@SuppressWarnings("unchecked")` 用来告诉编译器 **忽略类型转换的警告**，因为编译器无法在运行时验证泛型安全性

```java
/**
 * 将指定集合中的所有元素追加到此列表的末尾，
 * 顺序按指定集合迭代器返回的顺序。
 * 如果在操作进行过程中修改了指定集合，则此操作的行为未定义。
 * （注意，如果指定集合是此列表且非空，也会发生这种情况。）
 *
 * @param c 包含要添加到此列表的元素的集合
 * @return {@code true} 如果此列表因调用而发生变化
 * @throws NullPointerException 如果指定集合为 null
 */
public boolean addAll(Collection<? extends E> c) {
    return addAll(size, c);
}

/**
 * 将指定集合中的所有元素插入到此列表，从指定位置开始。
 * 将当前位于该位置的元素（如果有）以及之后的所有元素向右移动（索引加一）。
 * 新元素在列表中的顺序与指定集合迭代器返回的顺序一致。
 *
 * @param index 插入指定集合中第一个元素的位置索引
 * @param c 包含要添加到此列表的元素的集合
 * @return {@code true} 如果此列表因调用而发生变化
 * @throws IndexOutOfBoundsException {@inheritDoc}
 * @throws NullPointerException 如果指定集合为 null
 */
public boolean addAll(int index, Collection<? extends E> c) {
    checkPositionIndex(index);

    Object[] a = c.toArray();
    int numNew = a.length;
    if (numNew == 0)
        return false;

    Node<E> pred, succ;
    if (index == size) {
        succ = null;
        pred = last;
    } else {
        succ = node(index);
        pred = succ.prev;
    }

    for (Object o : a) {
        @SuppressWarnings("unchecked") E e = (E) o;
        Node<E> newNode = new Node<>(pred, e, null);
        if (pred == null)
            first = newNode;
        else
            pred.next = newNode;
        pred = newNode;
    }

    if (succ == null) {
        last = pred;
    } else {
        pred.next = succ;
        succ.prev = pred;
    }

    size += numNew;
    modCount++;
    return true;
}

```

### clear()

移除所有元素。

>1. 为什么不直接first = last = null; size = 0; modCount++; ，而是慢慢清空每个节点？
>
>**帮助垃圾回收器回收内存**
>
>- LinkedList 是链式结构，每个节点是一个对象，如果不把 `item/next/prev` 置 null，这些对象的引用还会互相连着，即使 `first/last` 被置 null，可能跨代 GC（老年代/新生代）不能及时回收。
>- 如果链很长，这些节点就会暂时占用内存，尤其是大列表。
>
>**防止潜在内存泄漏**
>
>- 即使我们已经没有直接访问链表的引用，如果链上元素里还引用了对象，这些对象就不会被 GC 回收。
>- 显式把每个节点的 `item`、`next`、`prev` 置 null，可以确保这些对象可以被回收。
>
>**对迭代器友好**
>
>- 如果有正在使用的迭代器，显式清理节点链接可以避免迭代器访问到旧节点时误用旧对象。

```java
/**
 * 移除此列表中的所有元素。
 * 调用返回后，列表将为空。
 */
public void clear() {
    // 清除节点之间的所有链接在技术上是“不必要”的，但：
    // - 如果被丢弃的节点跨越多个代，能帮助代际垃圾回收器（GC）
    // - 即使存在可达的迭代器，也能确保释放内存
    for (Node<E> x = first; x != null; ) {
        Node<E> next = x.next;
        x.item = null;
        x.next = null;
        x.prev = null;
        x = next;
    }
    first = last = null;
    size = 0;
    modCount++;
}
```

### indexOf()、lastIndexOf()

查找第一次出现的index, 如果找不到返回-1；

查找最后一次出现的index, 如果找不到返回-1；

```java
/**
 * 返回指定元素在此列表中第一次出现的索引，如果列表不包含该元素则返回 -1。
 * 更正式地说，返回最小索引 {@code i}，使得
 * <tt>(o==null ? get(i)==null : o.equals(get(i)))</tt> 成立，
 * 如果不存在这样的索引，则返回 -1。
 *
 * @param o 要搜索的元素
 * @return 指定元素在此列表中第一次出现的索引，如果列表不包含该元素则返回 -1
 */
public int indexOf(Object o) {
    int index = 0;
    if (o == null) {
        for (Node<E> x = first; x != null; x = x.next) {
            if (x.item == null)
                return index;
            index++;
        }
    } else {
        for (Node<E> x = first; x != null; x = x.next) {
            if (o.equals(x.item))
                return index;
            index++;
        }
    }
    return -1;
}

/**
 * 返回指定元素在此列表中最后一次出现的索引，如果列表不包含该元素则返回 -1。
 * 更正式地说，返回最大索引 {@code i}，使得
 * <tt>(o==null ? get(i)==null : o.equals(get(i)))</tt> 成立，
 * 如果不存在这样的索引，则返回 -1。
 *
 * @param o 要搜索的元素
 * @return 指定元素在此列表中最后一次出现的索引，如果列表不包含该元素则返回 -1
 */
public int lastIndexOf(Object o) {
    int index = size;
    if (o == null) {
        for (Node<E> x = last; x != null; x = x.prev) {
            index--;
            if (x.item == null)
                return index;
        }
    } else {
        for (Node<E> x = last; x != null; x = x.prev) {
            index--;
            if (o.equals(x.item))
                return index;
        }
    }
    return -1;
}
```

```java
}

/**
 * 获取但不移除此列表的最后一个元素，如果列表为空则返回 {@code null}。
 *
 * @return 此列表的最后一个元素，如果列表为空则返回 {@code null}
 * @since 1.6
 */
public E peekLast() {
    final Node<E> l = last;
    return (l == null) ? null : l.item;
}

/**
 * 获取并移除此列表的第一个元素，如果列表为空则返回 {@code null}。
 *
 * @return 此列表的第一个元素，如果列表为空则返回 {@code null}
 * @since 1.6
 */
public E pollFirst() {
    final Node<E> f = first;
    return (f == null) ? null : unlinkFirst(f);
}

/**
 * 获取并移除此列表的最后一个元素，如果列表为空则返回 {@code null}。
 *
 * @return 此列表的最后一个元素，如果列表为空则返回 {@code null}
 * @since 1.6
 */
public E pollLast() {
    final Node<E> l = last;
    return (l == null) ? null : unlinkLast(l);
}

/**
 * 将元素压入由此列表表示的栈。也就是说，将元素插入到列表开头。
 *
 * <p>此方法等价于 {@link #addFirst}。
 *
 * @param e 要压入的元素
 * @since 1.6
 */
public void push(E e) {
    addFirst(e);
}

/**
 * 从由此列表表示的栈中弹出元素。也就是说，移除并返回列表的第一个元素。
 *
 * <p>此方法等价于 {@link #removeFirst()}。
 *
 * @return 此列表开头的元素（栈顶元素）
 * @throws NoSuchElementException 如果列表为空
 * @since 1.6
 */
public E pop() {
    return removeFirst();
}

/**
 * 移除此列表中指定元素的第一次出现（从头到尾遍历）。
 * 如果列表不包含该元素，则列表保持不变。
 *
 * @param o 要从列表中移除的元素（如果存在）
 * @return {@code true} 如果列表包含指定元素
 * @since 1.6
 */
public boolean removeFirstOccurrence(Object o) {
    return remove(o);
}

/**
 * 移除此列表中指定元素的最后一次出现（从头到尾遍历）。
 * 如果列表不包含该元素，则列表保持不变。
 *
 * @param o 要从列表中移除的元素（如果存在）
 * @return {@code true} 如果列表包含指定元素
 * @since 1.6
 */
public boolean removeLastOccurrence(Object o) {
    if (o == null) {
        for (Node<E> x = last; x != null; x = x.prev) {
            if (x.item == null) {
                unlink(x);
                return true;
            }
        }
    } else {
        for (Node<E> x = last; x != null; x = x.prev) {
            if (o.equals(x.item)) {
                unlink(x);
                return true;
            }
        }
    }
    return false;
}
```

## 机制分析

### Fail-Fast机制

不难发现，LinkedList也有Fail-Fast机制，当我们进行如`add系列`、`remove系列`、`clear方法`等结构性修改的时候都有`modCount`的修改。

`LinkedList`的Fail-Fast机制与`ArrayLis`t相似，都是通过`modCount` + 迭代器检查实现的，详情见"ArrayList-机制分析-Fail-Fast机制"。

## 总结

`LinkedList` 是对 双向链表 的封装，实现了 `List` 和 `Deque` 接口，支持在头尾或中间高效插入、删除元素。逻辑简单直观，适合频繁在中间或头尾插入/删除元素的场景，但不适合需要大量随机访问的场景。

同时，与上篇的ArrayList对比，不难发现二者的异同：

**相同点**：

- 都实现了 Java 的 List 接口：它们都支持相同的基本操作，例如添加元素、删除元素、查找元素、获取元素等
- 都是有序的集合：都保证按顺序存储元素
- 都允许存储重复元素：可以有多个相同的元素，并且都允许存储 null 元素。
- 都不线程安全

**不同点**：

- **底层数据结构**：ArrayList的底层是动态数组，有扩容机制；LinkedList的底层是双向链表。
- **操作复杂度**：ArrayList的插入删除的复杂度，如果是头部和指定位置都是O(n)，如果是尾部插入，又分为需要扩容和不需要扩容的情况，如果需要扩容，那么会新建数组然后`copyOf()`，复杂度是O(n)，如果不需要扩容就是O(1)； LinkedList元素的头部/尾部的添加和删除都是O(1)，但是添加/删除中间节点的时候需要定位到节点，所以是O(n)。
- **快速随机访问**：`LinkedList` 不支持高效的随机元素访问，而 `ArrayList`（实现了 `RandomAccess` 接口） 支持。
- **内存空间占用**：`ArrayList` 的空间浪费主要体现在在 list 列表的结尾会预留一定的容量空间，而“LinkedList 每个节点额外存储前驱和后继引用，所以比 ArrayList 占用更多内存。
- **应用场景**：ArrayList 适合查询多，修改少的场景；LinkedList 适合插入和删除操作多的场景。
