`TreeMap` 基于红黑树实现 `NavigableMap`，键按 `Comparator` 或键类型的自然顺序全序排列。与 `HashMap` 的桶加链表或树不同，这里整棵搜索树既承担查找也承担有序导航。

本文摘录 **OpenJDK 8u** 中 `put`、节点结构、`fixAfterInsertion` 与删除相关入口，并简要对照 `TreeSet` 与 `TreeMap` 的关系。文中泛型已按阅读习惯补全。



<!-- more -->



## TreeMap 简介

`TreeMap` 使用一棵二叉搜索树存储键值对，并通过红黑着色与旋转保持近似平衡。`containsKey`、`put`、`remove` 的代价与树高同阶；红黑树保证树高随规模增长缓慢。

未指定 `Comparator` 时要求键实现 `Comparable`；指定比较器时由比较器定义顺序。`null` 键是否合法取决于比较器是否支持。

## 源码解析

### Entry 节点

每个节点保存键、值、左右孩子、父结点及颜色（红为 `false`，黑为 `true`，JDK8 用常量 `RED`/`BLACK` 表示）。

```java
static final class Entry<K, V> implements Map.Entry<K, V> {
    K key;
    V value;
    Entry<K, V> left;
    Entry<K, V> right;
    Entry<K, V> parent;
    boolean color = BLACK;

    Entry(K key, V value, Entry<K, V> parent) {
        this.key = key;
        this.value = value;
        this.parent = parent;
    }
    // getKey / getValue / setValue ...
}
```

### put：BST 插入与重平衡入口

空树时直接建根；否则沿根向下比较，左小右大，命中则覆盖 `value`；退出循环时把新节点挂在 `parent` 的左或右子，再调用 `fixAfterInsertion(e)`。

```java
public V put(K key, V value) {
    Entry<K, V> t = root;
    if (t == null) {
        compare(key, key);
        root = new Entry<>(key, value, null);
        size = 1;
        modCount++;
        return null;
    }
    int cmp;
    Entry<K, V> parent;
    Comparator<? super K> cpr = comparator;
    if (cpr != null) {
        do {
            parent = t;
            cmp = cpr.compare(key, t.key);
            if (cmp < 0)
                t = t.left;
            else if (cmp > 0)
                t = t.right;
            else
                return t.setValue(value);
        } while (t != null);
    } else {
        if (key == null)
            throw new NullPointerException();
        @SuppressWarnings("unchecked")
        Comparable<? super K> k = (Comparable<? super K>) key;
        do {
            parent = t;
            cmp = k.compareTo(t.key);
            if (cmp < 0)
                t = t.left;
            else if (cmp > 0)
                t = t.right;
            else
                return t.setValue(value);
        } while (t != null);
    }
    Entry<K, V> e = new Entry<>(key, value, parent);
    if (cmp < 0)
        parent.left = e;
    else
        parent.right = e;
    fixAfterInsertion(e);
    size++;
    modCount++;
    return null;
}
```

首行 `compare(key, key)` 在空树时用于触发对键类型与 `null` 的校验（自然顺序下空键会在此抛异常）。

### fixAfterInsertion：插入后着色与旋转

新插入结点先染红，若父也为红则违反「红不相邻」，按叔结点颜色分情况变色或旋转。实现与《算法导论》红黑树插入修正同构。

```java
private void fixAfterInsertion(Entry<K, V> x) {
    x.color = RED;
    while (x != null && x != root && x.parent.color == RED) {
        if (parentOf(x) == leftOf(parentOf(parentOf(x)))) {
            Entry<K, V> y = rightOf(parentOf(parentOf(x)));
            if (colorOf(y) == RED) {
                setColor(parentOf(x), BLACK);
                setColor(y, BLACK);
                setColor(parentOf(parentOf(x)), RED);
                x = parentOf(parentOf(x));
            } else {
                if (x == rightOf(parentOf(x))) {
                    x = parentOf(x);
                    rotateLeft(x);
                }
                setColor(parentOf(x), BLACK);
                setColor(parentOf(parentOf(x)), RED);
                rotateRight(parentOf(parentOf(x)));
            }
        } else {
            Entry<K, V> y = leftOf(parentOf(parentOf(x)));
            if (colorOf(y) == RED) {
                setColor(parentOf(x), BLACK);
                setColor(y, BLACK);
                setColor(parentOf(parentOf(x)), RED);
                x = parentOf(parentOf(x));
            } else {
                if (x == leftOf(parentOf(x))) {
                    x = parentOf(x);
                    rotateRight(x);
                }
                setColor(parentOf(x), BLACK);
                setColor(parentOf(parentOf(x)), RED);
                rotateLeft(parentOf(parentOf(x)));
            }
        }
    }
    root.color = BLACK;
}
```

### remove：deleteEntry 与 fixAfterDeletion

`remove` 先 `getEntry` 定位结点，再 `deleteEntry`。若被删结点有两棵子树，会用后继结点键值覆盖再删后继，与经典 BST 删除一致；然后根据被删结点颜色调用 `fixAfterDeletion` 恢复红黑性质。篇幅所限此处不展开整段 `deleteEntry`，阅读时可对照源码中「复制后继、断链、双黑修正」三块注释。

## TreeSet 与 TreeMap

`TreeSet` 实现 `NavigableSet`，内部持有 `NavigableMap<E, Object>`，默认即 `new TreeMap<>()`，元素作为 `key`，`PRESENT` 作为占位值：

```java
public class TreeSet<E> extends AbstractSet<E>
        implements NavigableSet<E>, Cloneable, java.io.Serializable {

    private transient NavigableMap<E, Object> m;
    private static final Object PRESENT = new Object();

    public TreeSet() {
        this(new TreeMap<>());
    }

    public boolean add(E e) {
        return m.put(e, PRESENT) == null;
    }
}
```

有序集合的导航方法（如 `lower`、`floor`、`pollFirst`）均委托给 backing `TreeMap` 的对应 `NavigableMap` API。

## 常见问题

### 1. TreeMap 与 HashMap 如何选型？

需要按键排序、`NavigableMap` 范围视图或稳定 `log n` 最坏代价时用 `TreeMap`。仅需要平均快速查找、不要求有序时用 `HashMap`。

### 2. compareTo 与 equals 不一致会怎样？

`TreeSet`/`TreeMap` 完全依赖比较结果判定相等，可能违反 `Set`/`Map` 一般契约，出现「逻辑上不相等却被视为同一键」等问题。比较规则应与 `equals` 一致或文档中明确说明差异。

### 3. 线程安全吗？

非线程安全，需外部同步或使用并发包中的有序替代方案。

-----------

## 总结

`TreeMap` 的核心是**红黑树维护的键有序映射**，`put` 走 BST 插入加 `fixAfterInsertion`，删除走 `deleteEntry` 与 `fixAfterDeletion`。

`TreeSet` 与 `TreeMap` 的关系和 `HashSet` 与 `HashMap` 相同，都是**有序键视图加占位值**；掌握 `TreeMap` 即掌握 `TreeSet` 的底层行为。
