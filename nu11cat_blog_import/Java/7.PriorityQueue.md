`PriorityQueue` 用数组表示一棵完全二叉堆，队头始终为当前最小元（由 `Comparable` 或 `Comparator` 定义「小」）。与 `TreeMap` 的红黑树不同，堆不维护全局有序遍历，只保证父子之间的偏序关系。

以下代码摘录自 **OpenJDK 8u**，泛型为阅读方便已补全。



<!-- more -->



## PriorityQueue 简介

类注释写明：`queue[n]` 的左右孩子分别为 `queue[2*n+1]` 与 `queue[2*(n+1)]`；若比较器为 `null`，则依赖元素自然顺序，且要求堆上每个结点不大于其任意后代。队列为无界语义，实际由数组容量伸缩承载。

不允许 `null` 元素。非线程安全，并发场景应使用 `PriorityBlockingQueue`。

## 源码解析

### 核心字段

```java
public class PriorityQueue<E> extends AbstractQueue<E>
        implements java.io.Serializable {

    private static final int DEFAULT_INITIAL_CAPACITY = 11;

    /**
     * 二叉堆数组：queue[n] 的孩子为 2n+1 与 2(n+1)。
     */
    transient Object[] queue;

    private int size = 0;
    private final Comparator<? super E> comparator;
    transient int modCount = 0;
}
```

### grow：扩容策略

容量小于 64 时近似翻倍并加 2，否则按原长度一半增长；随后 `Arrays.copyOf` 扩容数组。

```java
private void grow(int minCapacity) {
    int oldCapacity = queue.length;
    int newCapacity = oldCapacity + ((oldCapacity < 64) ?
            (oldCapacity + 2) :
            (oldCapacity >> 1));
    if (newCapacity - MAX_ARRAY_SIZE > 0)
        newCapacity = hugeCapacity(minCapacity);
    queue = Arrays.copyOf(queue, newCapacity);
}
```

### offer：在末尾插入后 siftUp

新元素先放在下标 `size` 处（堆尾），再通过 `siftUp` 与父结点比较并上浮，直到满足堆序。

```java
public boolean offer(E e) {
    if (e == null)
        throw new NullPointerException();
    modCount++;
    int i = size;
    if (i >= queue.length)
        grow(i + 1);
    size = i + 1;
    if (i == 0)
        queue[0] = e;
    else
        siftUp(i, e);
    return true;
}
```

### poll：弹出根后 siftDown

取出 `queue[0]`，把末尾元素挪到根位置，若队列非空则对该位置执行 `siftDown`，使结构重新满足堆性质。

```java
public E poll() {
    if (size == 0)
        return null;
    int s = --size;
    modCount++;
    E result = (E) queue[0];
    E x = (E) queue[s];
    queue[s] = null;
    if (s != 0)
        siftDown(0, x);
    return result;
}
```

### siftUp 与 siftDown

`siftUp` 沿父结点下标 `(k-1)>>>1` 上行，若当前键小于父结点则交换位置。`siftDown` 在存在左孩子时选取左右较小者与当前结点比较，不满足堆序则下沉。

```java
private void siftUp(int k, E x) {
    if (comparator != null)
        siftUpUsingComparator(k, x);
    else
        siftUpComparable(k, x);
}

@SuppressWarnings("unchecked")
private void siftUpComparable(int k, E x) {
    Comparable<? super E> key = (Comparable<? super E>) x;
    while (k > 0) {
        int parent = (k - 1) >>> 1;
        Object e = queue[parent];
        if (key.compareTo((E) e) >= 0)
            break;
        queue[k] = e;
        k = parent;
    }
    queue[k] = key;
}

private void siftDown(int k, E x) {
    if (comparator != null)
        siftDownUsingComparator(k, x);
    else
        siftDownComparable(k, x);
}

@SuppressWarnings("unchecked")
private void siftDownComparable(int k, E x) {
    Comparable<? super E> key = (Comparable<? super E>) x;
    int half = size >>> 1;
    while (k < half) {
        int child = (k << 1) + 1;
        Object c = queue[child];
        int right = child + 1;
        if (right < size &&
                ((Comparable<? super E>) c).compareTo((E) queue[right]) > 0)
            c = queue[child = right];
        if (key.compareTo((E) c) <= 0)
            break;
        queue[k] = c;
        k = child;
    }
    queue[k] = key;
}
```

### heapify：批量建堆

从最后一个非叶结点向前依次 `siftDown`，可在初始化集合内容后在线性时间内建立堆。

```java
private void heapify() {
    for (int i = (size >>> 1) - 1; i >= 0; i--)
        siftDown(i, (E) queue[i]);
}
```

## 常见问题

### 1. iterator 遍历是否有序？

`iterator()` 不保证按优先级顺序输出；需要有序遍历时应对副本排序或使用 `poll` 反复取出。

### 2. 与 TreeSet 取最小元的差异？

`PriorityQueue` 只快速访问堆顶；`TreeSet` 基于 `TreeMap` 维护全序结构，支持范围与导航操作，二者适用面不同。

### 3. remove(Object) 为何偏慢？

需在数组中线性查找再 `removeAt`，可能同时触发 `siftDown` 与 `siftUp` 以恢复堆。

## 总结

`PriorityQueue` 的本质是**数组二叉堆**：`offer` 走 `siftUp`，`poll` 走 `siftDown`，`grow` 负责扩容，`heapify` 处理批量初始化。

掌握比较器分支与自然顺序分支的对称实现后，阅读 `PriorityBlockingQueue` 等变体时可以直接对照堆修正路径。
