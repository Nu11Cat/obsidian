## ArrayList简介

`ArrayList`是Java集合`Collection`根接口下的一个实现类，实现了`List`接口，是顺序容器，即元素存放的数据与放进去的顺序相同，允许放入null元素，底层通过数组实现。ArrayList不是线程安全的。

## 源码解析

### 底层数据结构

可见ArrayList底层就是一个Object数组、一个size变量表示大小。

```java
/**
 * 用于存储 ArrayList 元素的数组缓冲区。
 * ArrayList 的容量就是此数组缓冲区的长度。任何
 * elementData == DEFAULTCAPACITY_EMPTY_ELEMENTDATA 的空 ArrayList
 * 在添加第一个元素时都会扩容到 DEFAULT_CAPACITY。
 */
transient Object[] elementData; // 非 private，以简化嵌套类的访问

/**
 * ArrayList 的大小（其包含的元素数量）。
 *
 * @serial
 */
private int size;
```

### 构造函数

1. **有参构造方法**中，如果传入列表的初始容量，那么就会构造相应大小的数组；但是如果传入0，则 `elementData` 指向 `EMPTY_ELEMENTDATA`，第一次 `add` 会触发扩容为 1，而不是懒加载的默认容量 10。；如果传入负数则会抛出异常。
2. **无参构造方法**中，构造了一个`ArrayList`，并不会真正初始化容量，而是将`elementData`指向`DEFAULTCAPACITY_EMPTY_ELEMENTDATA`，它又不同于有参构造方法中的`EMPTY_ELEMENTDATA`，这里的`DEFAULTCAPACITY_EMPTY_ELEMENTDATA`是一个信号，为`add`方法传达一个信号：当第一次存放元素的时候默认容量10。详情见下方“机制分析-懒加载”。
3. **集合构造器**用来把现有集合的所有元素复制到新的` ArrayList` 中，通过把数组作为桥梁进行转换。但是这里存在一个细节，`c.toArray() `可能返回的不是 Object[]，而是具体类型数组（比如 String[]）。ArrayList 内部要求 `elementData` 必须是 Object[]，所以用 Arrays.copyOf 拷贝一份为 Object[]。

```java
/**
 * 构造一个具有指定初始容量的空列表。
 *
 * @param  initialCapacity  列表的初始容量
 * @throws IllegalArgumentException 如果指定的初始容量为负数
 */
public ArrayList(int initialCapacity) {
    if (initialCapacity > 0) {
        this.elementData = new Object[initialCapacity];
    } else if (initialCapacity == 0) {
        this.elementData = EMPTY_ELEMENTDATA;
    } else {
        throw new IllegalArgumentException("Illegal Capacity: "+
                                           initialCapacity);
    }
}

/**
 * 构造一个初始容量为 10 的空列表。
 */
public ArrayList() {
    this.elementData = DEFAULTCAPACITY_EMPTY_ELEMENTDATA;
}

/**
 * 构造一个包含指定集合中元素的列表，
 * 元素顺序与该集合的迭代器返回它们的顺序相同。
 *
 * @param c 其元素将被放入此列表的集合
 * @throws NullPointerException 如果指定的集合为 null
 */
public ArrayList(Collection<? extends E> c) {
    elementData = c.toArray();
    if ((size = elementData.length) != 0) {
        // c.toArray 可能（错误地）不会返回 Object[]（参见 6260652）
        if (elementData.getClass() != Object[].class)
            elementData = Arrays.copyOf(elementData, size, Object[].class);
    } else {
        // 替换为空数组
        this.elementData = EMPTY_ELEMENTDATA;
    }
}
```

### 容量管理与扩容

1. **ensureCapacity**，是public，用于手动增加容量，保证至少能容纳 `minCapacity` 个元素。如果当前数组是默认空数组，就扩容至少为默认容量10，否则最小扩展容量为 0。
2. **ensureCapacityInternal**，是private，内部专用版本，add方法的懒加载就是在这里。详情见下方“机制分析-懒加载”。
3. **ensureExplicitCapacity**，是真正去判断是否需要扩容的方法。`modCount++`：记录结构修改，用于迭代器的快速失败检测。判断 `minCapacity` 是否大于当前数组长度，如果大于就调用 `grow(minCapacity)` 扩容。
4. **grow**，核心扩容逻辑。
   - `newCapacity = oldCapacity + oldCapacity/2` → 默认扩容 1.5 倍
   - 如果 1.5 倍后仍然小于 `minCapacity` → 扩容到 `minCapacity`
   - 如果超过最大数组长度 (`MAX_ARRAY_SIZE`) → 调用 `hugeCapacity(minCapacity)`
   - 最终用 `Arrays.copyOf` 生成新数组，并把旧数组复制过去
5. **hugeCapacity**，处理超大数组的边界情况。防止 JVM 分配过大的数组导致异常，同时最大可分配容量略小于 `Integer.MAX_VALUE`。

>**疑问：**
>
>1. `minExpand`是什么？
>
>它表示最小扩展容量，如果当前数组不是默认空数组 → `minExpand = 0`；否则则就`minExpand = DEFAULT_CAPACITY`（默认 10）。minExpand = 0 表示非默认数组，任意大小都可以；默认空数组则最小扩展容量为 DEFAULT_CAPACITY（10）。
>
>2. 为什么要设计两种 `ensureCapacity` 和 `ensureCapacityInternal`？
>
>ensureCapacity（public），给外部提供API，用于手动提前扩容
>
>ensureCapacityInternal（private），内部使用，由add调用，负责处理 懒加载默认容量 10（第一次 add）
>
>3. `modCount` 是什么，为什么在 `ensureExplicitCapacity` ++？
>
>`modCount`是结构修改计数器，ArrayList 内部维护，用于检测并发修改异常。当使用迭代器遍历时，如果发现 `modCount` 变化 → 抛出 `ConcurrentModificationException`。
>
>在`ensureExplicitCapacity`自增是因为扩容属于结构修改（内部数组替换），即使没有改变size也算修改了结构，所以必须自增modeCount来保证迭代器能检测到。
>
>4.  为什么扩容要控制在 `(minCapacity - MAX_ARRAY_SIZE)` 范围内？
>
>JVM 分配数组存在上限，一般为 `Integer.MAX_VALUE - 8`。令MAX_ARRAY_SIZE = Integer.MAX_VALUE - 8是为了防止扩容导致数组长度超过 JVM 可分配最大值 → OutOfMemoryError。

```java
/**
 * 如果必要，增加此 <tt>ArrayList</tt> 实例的容量，
 * 以确保它至少可以容纳由最小容量参数指定的元素数量。
 *
 * @param   minCapacity   所需的最小容量
 */
public void ensureCapacity(int minCapacity) {
    int minExpand = (elementData != DEFAULTCAPACITY_EMPTY_ELEMENTDATA)
        // 如果不是默认空数组，任何大小都可以
        ? 0
        // 如果是默认空数组，则最小扩展容量大于默认值。数组已应为默认大小。
        : DEFAULT_CAPACITY;

    if (minCapacity > minExpand) {
        ensureExplicitCapacity(minCapacity);
    }
}

private void ensureCapacityInternal(int minCapacity) {
    if (elementData == DEFAULTCAPACITY_EMPTY_ELEMENTDATA) {
        minCapacity = Math.max(DEFAULT_CAPACITY, minCapacity);
    }

    ensureExplicitCapacity(minCapacity);
}

private void ensureExplicitCapacity(int minCapacity) {
    modCount++;

    // 防溢出处理
    if (minCapacity - elementData.length > 0)
        grow(minCapacity);
}

/**
 * 数组可分配的最大大小。
 * 某些虚拟机会在数组中保留一些头部字。
 * 尝试分配更大的数组可能会导致
 * OutOfMemoryError: 请求的数组大小超出虚拟机限制
 */
private static final int MAX_ARRAY_SIZE = Integer.MAX_VALUE - 8;

/**
 * 增加容量，以确保它至少可以容纳由最小容量参数指定的元素数量。
 *
 * @param minCapacity 所需的最小容量
 */
private void grow(int minCapacity) {
    // 防溢出处理
    int oldCapacity = elementData.length;
    int newCapacity = oldCapacity + (oldCapacity >> 1); // 扩容1.5倍
    if (newCapacity - minCapacity < 0)
        newCapacity = minCapacity;
    if (newCapacity - MAX_ARRAY_SIZE > 0)
        newCapacity = hugeCapacity(minCapacity);
    // minCapacity 通常接近当前 size，所以这种扩容是合理的
    elementData = Arrays.copyOf(elementData, newCapacity);
}

private static int hugeCapacity(int minCapacity) {
    if (minCapacity < 0) // 溢出
        throw new OutOfMemoryError();
    return (minCapacity > MAX_ARRAY_SIZE) ?
        Integer.MAX_VALUE :
        MAX_ARRAY_SIZE;
}
```

### add()

1. **add(E e)**，将指定的元素追加到此列表的末尾。很简单，没啥好说的。

2. **add(int index, E element)**，在指定位置插入新元素，把后面的元素后移。其中`System.arraycopy(elementData, index, elementData, index + 1, size - index);`的参数分别表示

3. | 参数           | 作用                                             |
   | -------------- | ------------------------------------------------ |
   | `elementData`  | 源数组                                           |
   | `index`        | 从源数组的哪个位置开始复制（旧元素需要往后移动） |
   | `elementData`  | 目标数组（也是原数组）                           |
   | `index + 1`    | 目标位置（腾出插入位置）                         |
   | `size - index` | 要复制的元素个数（从 index 到 size-1）           |

>**疑问：**
>
>1. 为什么`add`的`modCount++;`写在`ensureCapacityInternal`，而不是直接写在方法里？
>
>问题在于`ensureCapacityInternal`的目的，它是`modCount`自增加上扩容判断，你固然可以把`ensureCapacityInternal`拆开，每个都像remove在外面写modCount自增，但是你还要写扩容判断。add(E e)是这样，addAll(...)是这样，add(int, E)也是，addAll(int, Collection)也是.......他们都要再写一次 `modCount++`，但是一旦你有一个漏写，整个ArrayList的fail-fast机制就报废了。再加上各种继承、批量插入、stream......风险太高了。所以我们直接把`ensureCapacityInternal`看作所有 add 的结构修改计数就好了。
>
>2. ArrayList 的尾部插入为什么是O(1)摊销，而不是严格O(1)？
>
>尾部插入不扩容是 O(1)，触发扩容是 O(n)，多次尾部插入的平均复杂度是摊销 O(1)。

```java
/**
 * 将指定的元素追加到此列表的末尾。
 *
 * @param e 要追加到此列表的元素
 * @return <tt>true</tt> （由 {@link Collection#add} 指定）
 */
public boolean add(E e) {
    ensureCapacityInternal(size + 1);  // 增加 modCount！！
    elementData[size++] = e;
    return true;
}

/**
 * 在此列表的指定位置插入指定元素。将当前位于该位置的元素（如果有）以及
 * 之后的所有元素向右移动（它们的索引加一）。
 *
 * @param index 要插入指定元素的位置索引
 * @param element 要插入的元素
 * @throws IndexOutOfBoundsException {@inheritDoc}
 */
public void add(int index, E element) {
    rangeCheckForAdd(index);

    ensureCapacityInternal(size + 1);  // 增加 modCount！！
    System.arraycopy(elementData, index, elementData, index + 1,
                     size - index);
    elementData[index] = element;
    size++;
}
```

### addAll

1. **addAll(Collection<? extends E> c)**，把指定集合 `c` 的所有元素追加到当前列表末尾。
2. **addAll(int index, Collection<? extends E> c)**，把指定集合 `c` 的所有元素 插入到指定位置 index。其中rangeCheckForAdd(index);是检验index合法性

```java
/**
 * 将指定集合中的所有元素追加到此列表的末尾，顺序按指定集合的迭代器返回的顺序。
 * 如果在操作进行过程中修改了指定集合，则此操作的行为未定义。
 * （这意味着如果指定集合是此列表且此列表非空，则调用行为未定义。）
 *
 * @param c 包含要添加到此列表的元素的集合
 * @return <tt>true</tt> 如果此列表因调用而发生变化
 * @throws NullPointerException 如果指定集合为 null
 */
public boolean addAll(Collection<? extends E> c) {
    Object[] a = c.toArray();
    int numNew = a.length;
    ensureCapacityInternal(size + numNew);  // 增加 modCount
    System.arraycopy(a, 0, elementData, size, numNew);
    size += numNew;
    return numNew != 0;
}

/**
 * 将指定集合中的所有元素插入到此列表，从指定位置开始。
 * 将当前位于该位置的元素（如果有）以及之后的所有元素向右移动（索引加一）。
 * 新元素在列表中的顺序与指定集合迭代器返回的顺序一致。
 *
 * @param index 插入指定集合中第一个元素的位置索引
 * @param c 包含要添加到此列表的元素的集合
 * @return <tt>true</tt> 如果此列表因调用而发生变化
 * @throws IndexOutOfBoundsException {@inheritDoc}
 * @throws NullPointerException 如果指定集合为 null
 */
public boolean addAll(int index, Collection<? extends E> c) {
    rangeCheckForAdd(index);

    Object[] a = c.toArray();
    int numNew = a.length;
    ensureCapacityInternal(size + numNew);  // 增加 modCount

    int numMoved = size - index;
    if (numMoved > 0)
        System.arraycopy(elementData, index, elementData, index + numNew,
                         numMoved);

    System.arraycopy(a, 0, elementData, index, numNew);
    size += numNew;
    return numNew != 0;
}
```

### remove()

1. **remove(int index)**，删除指定下标的元素，并前移后面的元素。
2. **remove(Object o)**，删除第一个匹配元素
3. **fastRemove(int index)** ，快速删除，代码复用且不返回旧值

>**疑问：**
>
>1. 为什么 `remove(Object)` 不能直接调用 `remove(index)`？而是又写了一个`fastRemove`？
>
>出于性能的考虑：1.我们不需要返回oldValue，使用remove会多进行一次数组访问；2.remove会rangeCheck(index);，但在 remove(Object) 里index 是遍历得出的，一定合法，多此一举。
>
>并且fastRemove是为 remove(Object) 设计的，remove(int index) 是给用户用的 API，代码职责应分离。
>
>2. 为什么`remove`的`modCount`变化要写在代码里面，而不是像add一样调用`ensureCapacityInternal`间接增加?
>
>问题在于`ensureCapacityInternal`的目的，它是`modCount`自增加上扩容判断，你固然可以把`ensureCapacityInternal`拆开，每个都像remove在外面写modCount自增，但是你还要写扩容判断。add(E e)是这样，addAll(...)是这样，add(int, E)也是，addAll(int, Collection)也是.......他们都要再写一次 `modCount++`，但是一旦你有一个漏写，整个ArrayList的fail-fast机制就报废了。再加上各种继承、批量插入、stream......风险太高了。所以我们直接把`ensureCapacityInternal`看作所有 add 的结构修改计数就好了。

```java
public E remove(int index) {
    // 检查下标
    rangeCheck(index);
    // 修改次数改变
    modCount++;
    // 获取旧的元素
    E oldValue = elementData(index);
    // 计算需要移动的下标（往前面移动一位）
    int numMoved = size - index - 1;
    if (numMoved > 0)
        // 调用native方法将后面的元素复制，移动往前一步
        System.arraycopy(elementData, index+1, elementData, index,
                         numMoved);
    // 把最后一个位置置 null，让垃圾回收方便进行
    elementData[--size] = null; // clear to let GC do its work

    return oldValue;
}


public boolean remove(Object o) {
    // 为空的元素
    if (o == null) {
        for (int index = 0; index < size; index++)
            if (elementData[index] == null) {
                fastRemove(index);
                return true;
            }
    } else {
        // 遍历，如果equals,则调用删除
        for (int index = 0; index < size; index++)
            if (o.equals(elementData[index])) {
                fastRemove(index);
                return true;
            }
    }
    return false;
}
// 快速删除方法
private void fastRemove(int index) {
    // 修改次数增加1
    modCount++;
    // 计算移动的位置
    int numMoved = size - index - 1;
    if (numMoved > 0)
        // 前面移动一位
        System.arraycopy(elementData, index+1, elementData, index,
                         numMoved);
    // 置空
    elementData[--size] = null; // clear to let GC do its work
}
```

### trimToSize()

把 ArrayList 的 容量（elementData.length） 调整为 当前 size

用来释放多余的内存，尤其在你添加/删除了很多元素后，数组可能比实际元素多很多空间。

```java
/**
 * 将此 <tt>ArrayList</tt> 实例的容量修剪为当前列表的大小。
 * 应用程序可以使用此操作来最小化 <tt>ArrayList</tt> 实例的存储空间。
 */
public void trimToSize() {
    modCount++;
    if (size < elementData.length) {
        elementData = (size == 0)
          ? EMPTY_ELEMENTDATA
          : Arrays.copyOf(elementData, size);
    }
}
```

### indexOf(), lastIndexOf()

```java
/**
 * 返回指定元素在此列表中首次出现的索引，如果列表不包含该元素则返回 -1。
 * 更正式地说，返回最小的索引 <tt>i</tt>，使得
 * <tt>(o==null ? get(i)==null : o.equals(get(i)))</tt> 成立，
 * 如果不存在这样的索引，则返回 -1。
 */
public int indexOf(Object o) {
    if (o == null) {
        for (int i = 0; i < size; i++)
            if (elementData[i]==null)
                return i;
    } else {
        for (int i = 0; i < size; i++)
            if (o.equals(elementData[i]))
                return i;
    }
    return -1;
}

/**
 * 返回指定元素在此列表中最后一次出现的索引，如果列表不包含该元素则返回 -1。
 * 更正式地说，返回最大的索引 <tt>i</tt>，使得
 * <tt>(o==null ? get(i)==null : o.equals(get(i)))</tt> 成立，
 * 如果不存在这样的索引，则返回 -1。
 */
public int lastIndexOf(Object o) {
    if (o == null) {
        for (int i = size-1; i >= 0; i--)
            if (elementData[i]==null)
                return i;
    } else {
        for (int i = size-1; i >= 0; i--)
            if (o.equals(elementData[i]))
                return i;
    }
    return -1;
}
```

## 机制分析

### 懒加载

ArrayList是懒加载的。

当我们使用`new ArrayList()`创建一个 `ArrayList` 时，并不会马上为它分配真正存储数据的数组空间，而是放了一个“占位的空数组”，并且在内部做了一个标记表示当第一次存放元素的时候默认容量10。当我们第一次执行`list.add()`，会一次性分配出长度为10的数组。

懒加载就是需要时才分配，节省资源。

从函数构造的代码中可以看到，当构造一个初始容量为 10 的空列表，实际上是elementData指向了DEFAULTCAPACITY_EMPTY_ELEMENTDATA：

```java
public ArrayList() {
    this.elementData = DEFAULTCAPACITY_EMPTY_ELEMENTDATA;	/here
}
```

然后当我们第一次执行`list.add()`，会判断是否等于`DEFAULTCAPACITY_EMPTY_ELEMENTDATA`：

```java
public boolean add(E e) {
    ensureCapacityInternal(size + 1);  //here
    elementData[size++] = e;
    return true;
}

private void ensureCapacityInternal(int minCapacity) {
    if (elementData == DEFAULTCAPACITY_EMPTY_ELEMENTDATA) {
        minCapacity = Math.max(DEFAULT_CAPACITY, minCapacity);	//here
    }

    ensureExplicitCapacity(minCapacity);
}
```

但是需要注意，如果是通过`ArrayList(int initialCapacity)`传入0构造ArraayList，则不会触发懒加载，因为不是`DEFAULTCAPACITY_EMPTY_ELEMENTDATA`，而是`EMPTY_ELEMENTDATA`，所以最终会触发`ensureExplicitCapacity`的`grow`，即大小扩容为1：

```java
public ArrayList(int initialCapacity) {
    if (initialCapacity > 0) {
        this.elementData = new Object[initialCapacity];	
    } else if (initialCapacity == 0) {
        this.elementData = EMPTY_ELEMENTDATA;	//here
    } else {
        throw new IllegalArgumentException("Illegal Capacity: "+
                                           initialCapacity);
    }
}

public boolean add(E e) {
    ensureCapacityInternal(size + 1);  //here
    elementData[size++] = e;
    return true;
}

private void ensureCapacityInternal(int minCapacity) {
    if (elementData == DEFAULTCAPACITY_EMPTY_ELEMENTDATA) {
        minCapacity = Math.max(DEFAULT_CAPACITY, minCapacity);
    }

    ensureExplicitCapacity(minCapacity);	//here
}

private void ensureExplicitCapacity(int minCapacity) {
    modCount++;

    if (minCapacity - elementData.length > 0)
        grow(minCapacity);					//here
}
```

### Fail-Fast机制

快速失败：当一个集合在迭代的过程中被 结构性修改时，迭代器会 立即抛出异常，而不是继续遍历可能已经被破坏的数据结构。只是一种快速检测机制，不保证线程安全。

在 Java 中：

- `ArrayList`、`HashMap` 等都实现了 Fail-Fast 机制
- 异常类型：`ConcurrentModificationException`
- 主要针对 单线程下的快速检测，而不是保证线程安全

ArrayList 的 Fail-Fast是通过核心字段`modCount `实现的：

```java
protected transient int modCount = 0;
```

`modCount` 记录集合的结构性修改次数。

核心逻辑是在Iterator 创建时会记录创建时的 modCount，称为 expectedModCount，在迭代过程中每次调用 next() 或 remove()，迭代器都会检查 modCount 是否与 expectedModCount 一致。不一致就抛异常 → 快速失败。

```java
//创建时
private int expectedModCount = modCount;

//迭代时
public E next() {
    checkForComodification();
    ...
}
private void checkForComodification() {
    if (modCount != expectedModCount)
        throw new ConcurrentModificationException();
}
```

### 扩容机制

add的系列操作都会进行`ensureCapacityInternal`，然后进行到`ensureExplicitCapacity`，即结构性修改次数++和扩容判断。当需要扩容的时候触发`grow`。

扩容逻辑：

- `newCapacity = oldCapacity + oldCapacity/2` → 默认扩容 1.5 倍
- 如果 1.5 倍后仍然小于 `minCapacity` → 扩容到 `minCapacity`
- 如果超过最大数组长度 (`MAX_ARRAY_SIZE`) → 调用 `hugeCapacity(minCapacity)`
- 最终用 `Arrays.copyOf` 生成新数组，并把旧数组复制过去

>1. 为什么扩容要控制在 `(minCapacity - MAX_ARRAY_SIZE)` 范围内？
>
>JVM 分配数组存在上限，一般为 `Integer.MAX_VALUE - 8`。
>
>令MAX_ARRAY_SIZE = Integer.MAX_VALUE - 8是为了防止扩容导致数组长度超过 JVM 可分配最大值导致OutOfMemoryError。

```java
private void ensureCapacityInternal(int minCapacity) {
    if (elementData == DEFAULTCAPACITY_EMPTY_ELEMENTDATA) {
        minCapacity = Math.max(DEFAULT_CAPACITY, minCapacity);
    }

    ensureExplicitCapacity(minCapacity);	//here
}

private void ensureExplicitCapacity(int minCapacity) {
    modCount++;

    if (minCapacity - elementData.length > 0)
        grow(minCapacity);		//here
}

private static final int MAX_ARRAY_SIZE = Integer.MAX_VALUE - 8;


private void grow(int minCapacity) {
    int oldCapacity = elementData.length;
    int newCapacity = oldCapacity + (oldCapacity >> 1); 	//here
    if (newCapacity - minCapacity < 0)
        newCapacity = minCapacity;
    if (newCapacity - MAX_ARRAY_SIZE > 0)
        newCapacity = hugeCapacity(minCapacity);
    elementData = Arrays.copyOf(elementData, newCapacity);		//here
}

private static int hugeCapacity(int minCapacity) {
    if (minCapacity < 0) 
        throw new OutOfMemoryError();
    return (minCapacity > MAX_ARRAY_SIZE) ?
        Integer.MAX_VALUE :
        MAX_ARRAY_SIZE;
}
```

## 总结

ArrayList 是典型的动态数组实现，通过懒加载、按需扩容和 Fail-Fast 迭代器，实现了随机访问高效、尾部操作快捷、内存可控的设计，同时也是理解 Java 集合框架设计思想和面试高频考点的经典案例。
